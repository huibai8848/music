## 1. 音乐管理模块

### 1.1 歌曲 CRUD

#### 数据表结构（song）

歌曲表存储每首歌曲的完整元数据，共计 17 列：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| title | VARCHAR(200) | 歌曲名 |
| artist_id | BIGINT | 主艺人 ID，关联 artist 表 |
| album_id | BIGINT | 所属专辑 ID（可为空） |
| duration | INT | 时长（秒） |
| audio_url | VARCHAR(255) | 音频文件路径 |
| cover_url | VARCHAR(255) | 封面图路径 |
| lyric_url | VARCHAR(255) | LRC 歌词文件路径 |
| lyrics | TEXT | 解析后的歌词 JSON |
| genre | VARCHAR(50) | 风格（流行/摇滚/古典等） |
| language | VARCHAR(20) | 语种（中文/英文/日文） |
| release_year | INT | 发行年份 |
| status | VARCHAR(20) | ACTIVE/PENDING/REJECTED |
| uploader_id | BIGINT | 上传者（会员上传时非空） |
| play_count | BIGINT | 播放量，默认 0 |
| created_time | DATETIME | 创建时间 |
| updated_time | DATETIME | 更新时间 |

#### 核心实现

**SongMapper.xml** — 分页查询列表（按创建时间降序）：

```sql
SELECT ... FROM song
<where>
  <if test="genre != null">AND genre = #{genre}</if>
  <if test="language != null">AND language = #{language}</if>
  <if test="releaseYear != null">AND release_year = #{releaseYear}</if>
  <if test="artistId != null">AND artist_id = #{artistId}</if>
  <if test="albumId != null">AND album_id = #{albumId}</if>
  <if test="status != null">AND status = #{status}</if>
</where>
ORDER BY created_time DESC
LIMIT #{limit} OFFSET #{offset}
```

**SongServiceImpl.enrichSongVO()** — 查询歌曲后自动填充关联数据：

```
① 根据 artistId 查询艺人名 → 设置 artistName
② 根据 albumId 查询专辑名 → 设置 albumTitle
③ 查询 song_category 关联 → 设置 categoryIds（分类 ID 列表）
④ 查询 likes 表 → 设置 likeCount（喜欢数）
⑤ 查询 favorite 表 → 设置 favoriteCount（收藏数）
```

每个查询独立 try-catch，单个字段失败不影响整体。

#### 接口方法

| 方法 | URL | 鉴权 | 说明 |
|------|-----|------|------|
| GET | /api/songs | 公开 | 分页列表（支持 6 维筛选） |
| GET | /api/songs/{id} | 公开 | 歌曲详情（含关联艺人/专辑名） |
| GET | /api/songs/{id}/lyrics | 公开 | LRC 歌词 JSON |
| GET | /api/songs/{id}/categories | 公开 | 歌曲分类 ID 列表 |
| PUT | /api/songs/{id}/categories | Admin | 更新歌曲分类 |
| POST | /api/songs | Admin | 创建歌曲 |
| PUT | /api/songs | Admin | 更新歌曲 |
| DELETE | /api/songs/{id} | Admin | 删除歌曲（同步清除缓存） |

**删除/更新时的缓存清理**：
- 热门歌曲缓存 `cache:hot_songs`（每次更新/删除时清除）
- 歌词缓存 `cache:lyrics:{songId}`（删除时清除）

### 1.2 专辑聚合管理

#### 数据表结构（album）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| title | VARCHAR(200) | 专辑名 |
| artist_id | BIGINT | 所属艺人 ID |
| cover_url | VARCHAR(255) | 封面图 URL |
| description | TEXT | 专辑简介 |
| release_date | DATE | 发行日期 |
| created_time / updated_time | DATETIME | 时间戳 |

#### 核心实现

**AlbumController** — 专辑的公开浏览接口：

```
GET /api/albums        → 分页列表（支持标题模糊搜索）
GET /api/albums/search → 按专辑名模糊搜索
GET /api/albums/{id}   → 详情（含该专辑下所有歌曲列表）
```

**专辑详情页的数据聚合**：

```
① 根据专辑 ID 查询 Album 实体
② 查询该专辑下所有 status = ACTIVE 的歌曲（songMapper.selectByAlbumId）
③ 每首歌曲通过 enrichSongVO 填充艺人名等关联信息
④ 返回给前端的 VO 中包含专辑信息和歌曲列表两个维度
```

**边界处理**：
- 专辑被删除时，关联歌曲的 `album_id` 置为 NULL（SET NULL 策略）
- 专辑创建时自动关联艺人，无需额外关联表
- 专辑搜索仅按 `title` 字段匹配（LIKE %keyword%）

### 1.5 会员上传与审核

#### 整体流程

```
会员上传 → PENDING(审核中) → 管理员审核 → ACTIVE(上架) / REJECTED(驳回)
   ↑                             |
   └── 通知上传者审核中          └── 通知上传者审核结果
```

#### 核心实现（MemberController）

**会员上传接口** `POST /api/members/upload-music`：

```
① 校验身份
   - role = VIP 或 ADMIN
   - vip_expire_time 未过期

② 校验配额
   - 每日上传次数（Redis 计数，默认 5 次/天）
   - 存储空间（Redis 追踪，VIP 上限 500MB）

③ 文件处理
   - 接收 multipart 请求：音频 + 封面 + 歌词 + songInfo JSON
   - 音频写入 audio/{yyyy}/{MM}/{uuid}.mp3
   - 封面写入 cover/{yyyy}/{MM}/{uuid}.jpg
   - 歌词写入 lyric/{yyyy}/{MM}/{uuid}.lrc
   - 使用 Apache Tika 提取音频时长

④ 元数据解析
   - 解析 songInfo JSON → 构建 Artist/Album/Song 实体
   - 自动映射 genre/language/releaseYear → category 分类 ID
   - 事务性写入数据库（Artist → Album → Song 原子提交）

⑤ 成功后处理
   - 递增上传计数（Redis）
   - 更新存储使用量（Redis）
   - 管理员上传 → status = ACTIVE（直接上架）
   - 会员上传 → status = PENDING（需审核）
```

**审核接口** `PUT /api/admin/songs/{id}/audit`：

```
① 乐观锁更新（仅 PENDING 状态可被审核，防并发重复审核）
   UPDATE song SET status = #{status} WHERE id = #{id} AND status = 'PENDING'

② 审核通过（ACTIVE）
   - 清除热门歌曲缓存
   - 通知上传者：歌曲已上架

③ 审核驳回（REJECTED）
   - 通知上传者：驳回原因
```

#### SongService.submitSongForReview 事务流程

提交审核时在一个事务中顺序完成：

```
① resolveArtistForSubmit()
   - 已有 ID → 校验存在
   - 按名称查找 → 存在则复用
   - 不存在 → INSERT 新建艺人

② resolveAlbumForSubmit()
   - 已有 ID → 校验存在（无效不报错）
   - 按标题+艺人查找 → 存在则复用
   - 不存在 → INSERT 新建专辑

③ INSERT song（playCount = 0，status = PENDING）

④ 保存 song_category 关联（如提供分类 ID）

⑤ 解析 LRC 歌词 → 回写 lyrics 字段

⑥ 延迟（事务提交后）通知上传者 + 通知所有管理员
```

#### 边界条件

| 场景 | 处理 |
|------|------|
| 非会员调用上传 | 返回 5002（需会员） |
| 上传次数超限 | 返回 5003 |
| 存储空间不足 | 返回 5004 |
| 音频格式非 MP3/AAC | 返回 4001 |
| 歌曲已被审核 | 乐观锁防止重复审核 |
| 上传失败消耗次数？ | 不消耗（次数在成功后递增） |

## 2. 搜索发现模块

### 2.1 全局模糊搜索

#### 核心实现（SongServiceImpl.searchSongs）

**接口**：`GET /api/songs/search?q=关键字&page=1&size=20`

**SQL 实现（SongMapper.xml）**：

```sql
SELECT s.* FROM song s
LEFT JOIN artist a ON s.artist_id = a.id
LEFT JOIN album al ON s.album_id = al.id
WHERE s.status = 'ACTIVE'
  AND (s.title LIKE CONCAT('%', #{keyword}, '%')
       OR a.name LIKE CONCAT('%', #{keyword}, '%')
       OR al.title LIKE CONCAT('%', #{keyword}, '%'))
ORDER BY
    CASE
        WHEN s.title LIKE CONCAT('%', #{keyword}, '%') THEN 0
        WHEN a.name LIKE CONCAT('%', #{keyword}, '%') THEN 1
        ELSE 2
    END,
    s.play_count DESC
LIMIT #{limit} OFFSET #{offset}
```

**搜索逻辑**：

```
① 接收 q 参数（歌曲名/艺人名/专辑名），判空直接返回空列表
② SQL LIKE %q% 三表关联模糊匹配（song LEFT JOIN artist + album）
③ 相关性排序：歌曲名匹配 > 艺人名匹配 > 专辑名匹配
④ 同等优先级按播放量（play_count DESC）二次排序
⑤ 仅搜索 status = ACTIVE 的上架歌曲
⑥ 分页返回，默认 20 条/页
```

**搜索结果缓存**：MD5(query) 作为 Redis key，TTL = 60s

**前端实现**（Search.vue）：
```
① 输入框防抖（300ms 延迟触发搜索）
② 搜索类型切换：歌曲 / 艺人 / 专辑
③ 搜索结果分页展示，支持滚动加载
```

**边界条件**：
| 场景 | 处理 |
|------|------|
| 搜索关键字为空 | 返回空列表 |
| 无匹配结果 | 返回空列表 |
| 搜索过于频繁 | 无限制（公开接口） |

### 2.4 播放排行榜单

#### 核心实现（RankingController）

**接口**：`GET /api/rankings?type=all&limit=50`

**统一查询逻辑**（日榜/周榜/月榜/总榜均相同）：

```sql
SELECT ... FROM song WHERE status = 'ACTIVE'
ORDER BY play_count DESC
LIMIT #{limit}
```

四种 `type` 参数仅返回的 `label` 不同：
| type | label | 说明 |
|------|-------|------|
| daily | 日榜 | 当日排行（与周/月/总榜数据源相同，均按 play_count DESC） |
| weekly | 周榜 | 本周排行 |
| monthly | 月榜 | 本月排行 |
| all | 总榜 | 历史总排行 |

**播放量统计方式**（PlayHistoryService.recordPlay）：

```
① 前端播放歌曲 → POST /api/history { songId }
② 后端在一个事务中：
   - INSERT play_history（记录用户播放行为）
   - UPDATE song SET play_count = play_count + 1（实时递增）
③ 排行榜直接从 song.play_count 读取，无需额外计算
```

**前端 Rankings.vue 实现**：

```
① Tab 切换：日榜 / 周榜 / 月榜 / 总榜
② 歌曲列表展示：排名序号 + 封面图 + 歌名 + 艺人 + 播放量
③ 排名前 3 高亮显示（红色序号）
④ 双击行 → 自动播放该歌曲
⑤ 点击歌名 → 跳转歌曲详情页
```

**边界条件**：

| 场景 | 处理 |
|------|------|
| 无数据 | 显示"暂无数据" |
| 加载中 | 显示"加载中..." |
| type 参数无效 | 返回 400 错误 |

### 2.5 最近播放记录

#### 核心实现（PlayHistoryController）

| 方法 | URL | 说明 |
|------|-----|------|
| GET | /api/history | 获取最近播放记录（最多 100 条） |
| POST | /api/history | 记录播放历史 |
| DELETE | /api/history | 清除播放记录 |

**记录流程**（PlayHistoryServiceImpl.recordPlay）：

```
① 插入 play_history 表（user_id, song_id, played_time = NOW()）
② 递增 song.play_count（播放量统计）
③ 检查该用户记录数是否超过 100 条
④ 超过 → 删除最旧的超出部分（deleteOldRecords）
```

**查询流程**（PlayHistoryServiceImpl.getHistory）：

```
① 根据 userId 查询最近 100 条记录（按播放时间降序）
② 关联查询每首歌曲的信息（title, coverUrl, duration）
③ 如果歌曲已被删除，显示为"歌曲已删除"
```

**前端记录时机**（playback.js）：

```
① 每次切换歌曲时触发（recordPlayHistory）
② 通过 POST /api/history 异步上报（不阻塞播放）
③ 播放历史页面展示：歌曲封面 + 歌名 + 播放时间
```

**前端播放计数上报**（`POST /api/songs/{id}/play`）：

```
历史接口，保留空实现避免前端调用报错。
计数逻辑已合入 POST /api/history 流程中。
```

**边界条件**：

| 场景 | 处理 |
|------|------|
| 歌曲已删除 | 显示"歌曲已删除" |
| 播放记录超 100 条 | 自动清理最旧记录 |
| 未登录调用 | 401 需要先登录 |
| 清除记录 | DELETE 操作不可恢复 |

## 3. 音乐播放模块

### 3.1 全局音频播放器

#### 架构设计

**前端 Pinia Store（playback.js）** 管理全局播放状态：

```
PlaybackState:
├── currentSong: Song | null       // 当前播放歌曲
├── queue: Song[]                  // 播放队列
├── queueIndex: number             // 队列当前位置
├── mode: PlayMode                 // 播放模式
├── volume: number  (0-1)          // 音量
├── progress: number (ms)          // 当前播放进度
├── duration: number (ms)          // 总时长
├── isPlaying: boolean             // 播放状态
└── isMuted: boolean               // 静音状态
```

#### 组件层次


```javascript
playSong(song)         // 播放指定歌曲
```

```
① 用户双击歌曲 → playSong(song) 或 replaceQueue(list, index)

④ audio.src = song.audioUrl → audio.play()
⑤ audio 事件监听：timeupdate → 更新 progress
⑥ audio 事件监听：ended → 自动下一首
⑦ 错误处理：加载失败 → 自动播放下首
```

### 3.2 LRC 歌词同步

#### 后端 LRC 解析（SongServiceImpl.parseLrcFile）

**输入**：LRC 文件路径（`lyric/{yyyy}/{MM}/{uuid}.lrc`）

**输出**：JSON 格式歌词数组

```json
[
  {"time": 12.5, "text": "想留不能留 才最寂寞"},
  {"time": 16.8, "text": "没说完温柔 只剩离歌"},
  ...
]
```

**解析流程**：

```
① 读取 LRC 文件（UTF-8 编码）
② 正则匹配 [mm:ss.xx] 时间戳
③ 同一行支持多个时间戳（[00:01.00][00:05.00]歌词）
④ 过滤元信息行：[ti:], [ar:], [al:], [by:], [offset:]
⑤ 按时间排序后序列化为 JSON
```

**缓存**：解析结果写入 `cache:lyrics:{songId}`，TTL = 1800s

**接口**：`GET /api/songs/{id}/lyrics`

#### 前端歌词同步（LyricsPanel.vue）

```
① requestAnimationFrame 动画循环驱动（60fps）
② 根据 Audio.currentTime 二分查找当前歌词索引
③ 高亮当前行（.lyric-line.active）
④ 提前 2 行滚动容器（scrollTop 平滑滚动）
⑤ 支持点击歌词行 → seekTo(该行时间戳)
⑥ 无歌词 → 显示"暂无歌词"
```

#### 队列数据结构

### 3.4 播放模式切换

#### 四种播放模式

| 模式 | 行为 | 图标 |
|------|------|------|
| SEQUENCE | 顺序播放，队列末尾自动停止 | → |
| LOOP | 列表循环，末尾回到开头 | 🔄 |
| REPEAT_ONE | 单曲循环 | 🔂 |
| SHUFFLE | 随机播放（已播标记防重复） | 🔀 |

#### 循环逻辑实现

```javascript
function nextSong() {
  switch (mode.value) {
    case 'SEQUENCE':
      if (queueIndex.value < queue.value.length - 1) {
        queueIndex.value++  // 下一首
      } else {
        stopPlaying()       // 队列末尾停止
      }
      break
    case 'LOOP':
      queueIndex.value = (queueIndex.value + 1) % queue.value.length
      break
    case 'REPEAT_ONE':
      // queueIndex 不变，重新播放当前曲
      break
    case 'SHUFFLE':
      // 从未播放过的歌曲中随机选一首
      // 所有歌曲都已播放 → 重置已播标记
      do {
        nextIdx = randomIndex(queue.value.length)
      } while (playedSet.has(nextIdx) && playedSet.size < queue.value.length)
      queueIndex.value = nextIdx
      break
  }
}
```

**SHUFFLE 模式细节**：
- `playedSet` 记录已播放的队列索引
- 播放完所有歌曲后自动重置（重新洗牌）
- 切换到其他模式时不清除 playedSet

### 3.5 进度记忆续播

#### 后端存储

| 接口 | 方法 | 说明 |
|------|------|------|
| POST /api/progress | 上报进度 | body: { songId, progress(ms) } |
| GET /api/progress/{songId} | 查询进度 | 返回 progress(ms) |

**Redis 存储结构**：
- Key: `progress:{userId}:{songId}`
- Value: `"45000"`（ms 字符串）
- TTL: 7 天

#### 前端流程

```
① 播放中 → 每 10 秒上报一次进度（PlayerBar 定时器驱动）
② 切换歌曲时 → 立即上报当前歌曲进度
③ 开始播放新歌 → 查询 GET /api/progress/{songId}
④ 有进度 > 5s → 弹窗提示
⑤ 用户选择"是" → audio.currentTime = savedProgress
⑥ 用户选择"否" → 从头播放
```

#### 边界条件

| 场景 | 处理 |
|------|------|
| 进度 < 5s | 不提示续播 |
| 歌曲第一次播放 | 进度为 null，不提示 |
| 7天内未再播放 | 进度自动过期 |

### 3.6 黑胶碟片动画

#### 组件实现（DiscAnimation.vue）

**CSS 动画**：

```css
.disc {
  width: 200px;
  height: 200px;
  border-radius: 50%;
  background: radial-gradient(circle at center, #333 0%, #111 30%, #222 60%, #444 100%);
  animation: spin 8s linear infinite;
  animation-play-state: paused;  /* 暂停时停止旋转 */
}
.disc.playing {
  animation-play-state: running; /* 播放时旋转 */
}
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

**效果**：
- 黑胶唱片风格：中心圆形标签 + 黑色盘面 + 同心圆纹路
- 封面以圆形裁切显示在碟片中心
- 播放时匀速旋转（8 秒一圈）
- 暂停时停止旋转（保持当前角度）
- 悬停时显示歌曲信息浮层

#### 集成位置

```
PlayerBar 展开歌词面板时 → 左侧显示黑胶碟片
歌曲详情页 → 大尺寸黑胶碟片作为背景装饰
```
