## 一、原有需求盘点（已覆盖）

| 模块 | 覆盖内容 | 完整度 |
|------|---------|--------|
| 用户 | 注册/登录/登出、角色区分、个人主页背景、会员充值 | ⚠️ 缺少密码安全、Token 鉴权 |
| 音乐播放 | 全局播放器、进度条、封面碟片动画、LRC 歌词同步 | ✅ 较完整 |
| 交互 | 收藏/喜欢、评论、歌单 CRUD | ✅ 较完整 |
| 会员特权 | 歌房(双人同步听歌)、聊天+语音、个人音乐库 500MB | ⚠️ 缺少实时同步细节 |
| 管理员 | 音乐导入、图片管理、统计表单、封号、删评 | ⚠️ 缺少操作日志 |

## 三、Redis 集成设计（核心补充）

### 3.1 Redis 使用场景总览

```
┌─────────────────────────────────────────┐
│              Redis 服务                  │
├─────────────────────────────────────────┤
│ 1. Session / Token 存储                 │
│    - 黑名单 Token（登出后失效）          │
│    - refresh_token 管理                 │
│                                         │
│ 2. 缓存层                               │
│    - 热门歌曲列表（缓存 5 分钟）         │
│    - 歌单详情（缓存 3 分钟）             │
│    - LRC 歌词解析结果（缓存 30 分钟）     │
│    - 首页推荐数据（缓存 10 分钟）        │
│                                         │
│ 3. 实时状态                             │
│    - 歌房状态（当前歌曲、进度、状态）    │
│    - 用户在线状态                        │
│    - 播放进度临时存储                    │
│                                         │
│ 4. 计数与限流                           │
│    - 播放计数缓冲（定时刷入 DB）         │
│    - API 调用频率限制（滑动窗口）         │
│    - 每日上传次数限制                    │
│                                         │
│ 5. 消息队列 / Pub-Sub                   │
│    - 歌房操作同步广播                    │
│    - 系统通知推送                        │
│    - 异步任务触发                        │
└─────────────────────────────────────────┘
```

### 3.2 关键数据结构设计

```redis
# Token 黑名单（TTL = token 剩余有效期）
BL_token:{jwt_id} -> "1" (TTL)

# 缓存：热门歌曲列表（TTL 300s）
cache:hot_songs -> JSON [{id, title, artist, cover}]

# 缓存：歌词解析（TTL 1800s）
cache:lyrics:{song_id} -> JSON [{time, text}]

# 歌房状态（Hash）
room:{room_id} -> {
    current_song_id: 123,
    status: "playing"|"paused",
    progress: 45000,  # ms
    owner_id: 1,
    member_ids: [2, 3],
    play_mode: "sequence"|"repeat"|"random"
}

# 播放计数缓冲（Sorted Set）
play_count:buffer -> { song_id: count }
# 定时任务将 count 累加到 DB，然后清空

# API 限流（滑动窗口，TTL 60s）
ratelimit:{user_id}:{api_path} -> count (TTL 60)

# 用户在线状态（String，TTL 300s，心跳续期）
online:{user_id} -> "1" (TTL 300)
```

## 四、部署方案调整（Ngrok + 本地服务）

### 4.1 架构示意

```
┌─────────────────────────────────────┐
│          Windows 开发机              │
│  ┌──────────┐  ┌──────────────────┐ │
│  │ 前端      │  │ 后端 Spring Boot │ │
│  │ Vue 3    │◄─┤ Java 17          │ │
│  │ Nginx    │  │ WebSocket        │ │
│  └────┬─────┘  └───────┬──────────┘ │
│       │                │            │
│  ┌────┴────────────────┴──────────┐ │
│  │        Redis (本地)            │ │
│  └───────────────────────────────┘ │
│  ┌───────────────────────────────┐ │
│  │        MySQL (本地)           │ │
│  └───────────────────────────────┘ │
│  ┌───────────────────────────────┐ │
│  │   Ngrok 隧道                  │ │
│  │   http://xxx.ngrok.io ───►   │ │
│  │   转发到 localhost:8080       │ │
│  └───────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 4.2 Ngrok 配置要点

- 注册 Ngrok 账号，获取 authtoken
- 配置 `ngrok.yml`，固定域名（paid plan）或随机域名（free）
- 同时暴露 HTTP(80) 和 WebSocket 端口
- 前端 API 基址根据环境自动切换（开发=localhost，生产=ngrok 域名）
- Ngrok Dashboard 查看实时请求状态

### 4.3 服务启动顺序

```
MySQL → Redis → 后端 Spring Boot → 前端 Nginx → Ngrok
```

| 文件类型 | 存储位置 | 访问方式 | 目录结构 |
|---------|---------|---------|---------|
| 音频 (MP3) | 本地磁盘 | 静态资源映射 / 流式传输 | `/data/music/audio/{yyyy/mm}/{filename}` |
| 封面图片 | 本地磁盘 | 静态资源映射 | `/data/music/cover/{yyyy/mm}/{filename}` |
| 歌词 (LRC) | 本地磁盘 | 静态资源映射 | `/data/music/lyric/{yyyy/mm}/{filename}` |
| 用户头像 | 本地磁盘 | 静态资源映射 | `/data/music/avatar/{yyyy/mm}/{filename}` |
| 背景图片 | 本地磁盘 | 静态资源映射 | `/data/music/background/{yyyy/mm}/{filename}` |
| 语音消息 | 本地磁盘 | 临时 + 定期清理 | `/data/music/voice/{room_id}/{filename}` |

> 所有上传接口统一使用 `/api/files/upload` 封装，返回文件访问 URL。

## 九、开发优先级建议

```
第一优先级（MVP，可运行的基本平台）
├── 阶段一：项目初始化与基础设施
├── 阶段二：数据库设计与初始化
├── 阶段三：Redis 集成
├── 阶段四：用户模块（注册/登录/JWT）
├── 阶段五：音乐管理（管理员导入+搜索）
├── 阶段六：音乐播放（全局播放器+歌词）
├── 阶段七：交互模块（评论/收藏/歌单）
└── 部署：Ngrok + Nginx 使外网可访问

第二优先级（会员增值功能）
├── 阶段八：歌房功能（WebSocket 同步）
├── 会员个人音乐库上传
└── 会员标识展示

第三优先级（运营与管理完善）
├── 阶段九：管理员后台完善
├── 通知系统
├── 举报系统
├── 排行榜/统计
└── 操作日志

第四优先级（体验优化）
├── 播放进度记忆
├── 响应式适配
├── 密码重置（邮箱）
├── 批量导入
└── 数据备份自动化
```
