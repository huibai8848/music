## 2. 用户模块

### 2.1 功能概述

用户注册、登录、鉴权、个人信息管理、会员体系。是整个平台的认证与授权基础。

### 2.2 数据结构

```
User 实体
├── id: Long (PK)
├── email: String (UNIQUE)       // 登录账号，邮箱格式
├── password: String              // BCrypt 加密密文
├── nickname: String              // 昵称
├── avatar: String                // 头像 URL
├── bio: String                   // 个人简介
├── background: String            // 个人主页背景图 URL
├── role: Enum(USER, ADMIN, VIP) // 角色
├── vipExpireTime: LocalDateTime  // 会员过期时间（null 表示非会员）
├── status: Enum(ACTIVE, BANNED) // 状态
├── createdTime: LocalDateTime
└── updatedTime: LocalDateTime
```

### 2.3 接口方法

| 方法 | URL | 说明 | 鉴权 |
|------|-----|------|------|
| `register` | POST /api/auth/register | 邮箱+密码注册 | 无 |
| `login` | POST /api/auth/login | 登录，返回 JWT | 无 |
| `refreshToken` | POST /api/auth/refresh | 刷新 access_token | RefreshToken |
| `logout` | POST /api/auth/logout | 登出，拉黑 Token | JWT |
| `getCurrentUser` | GET /api/users/me | 获取当前用户信息 | JWT |
| `updateProfile` | PUT /api/users/me | 修改昵称/头像/简介/背景 | JWT |
| `changePassword` | PUT /api/users/me/password | 旧密码验证修改 | JWT |
| `getPublicProfile` | GET /api/users/{id} | 查看他人公开主页 | 可选 JWT |
| `rechargeMembership` | POST /api/users/me/membership | 充值会员 | JWT |

### 2.4 实现逻辑

#### 2.4.1 注册流程

```
① 校验 email 格式 + 密码强度（8-20位含字母数字）
② BCrypt.hashpw(password) 加密密码
③ 检查 email 是否已注册（防重复）
④ INSERT user 记录
⑤ 返回成功
```

#### 2.4.2 登录流程

```
① 根据 email 查询用户
② BCrypt.checkpw(password, user.password) 校验密码
③ 生成 JWT：access_token（2h 过期，含 userId/role）
④ 生成 refresh_token（7d 过期），存入 Redis
⑤ 返回 tokens + 用户信息
```

#### 2.4.3 Token 刷新

```
① 校验 refresh_token 有效性
② 检查 Redis 中是否存在该 refresh_token（是否已被吊销）
③ 生成新的 access_token（旧的访问期仍有效，不主动失效）
④ 返回新 access_token
```

#### 2.4.4 登出

```
① 解析当前 access_token 的 jti（JWT ID）
② 将 jti 加入 Redis 黑名单，TTL = token 剩余有效期
③ 删除 refresh_token
④ 返回成功
```

### 2.5 边界条件

| 边界 | 处理方式 |
|------|---------|
| 重复注册同一邮箱 | 返回 1001(USER_ALREADY_EXISTS) |
| 密码错误 5 次 | 暂时锁定 15 分钟（Redis 计数） |
| Token 过期 | 返回 401，前端调用 refresh 接口 |
| Token 在黑名单中 | 返回 401，要求重新登录 |
| 被封禁用户登录 | 返回 1004(USER_BANNED) |
| 会员已过期 | 正常登录，部分接口限制使用 |
| refresh_token 过期 | 返回 401，前端跳转登录页 |

### 2.6 测试用例

| 编号 | 测试场景 | 输入 | 预期 |
|------|---------|------|------|
| UT-01 | 注册成功 | email: a@b.com, pwd: Abc12345 | 200 + userId |
| UT-02 | 注册重复邮箱 | 同上邮箱再注册 | 400, code=1001 |
| UT-03 | 密码强度不足 | pwd: 123 | 422 参数校验失败 |
| UT-04 | 登录成功 | 正确凭证 | 200 + access_token + refresh_token |
| UT-05 | 登录密码错误 | 密码错误 | 200, code=1002 |
| UT-06 | 被封禁用户登录 | 封禁用户凭证 | 200, code=1004 |
| UT-07 | 刷新 Token | 有效 refresh_token | 200 + 新 access_token |
| UT-08 | 登出后再用旧 Token | 已登出的 access_token | 401 |

## 3. 音乐管理模块

### 3.1 功能概述

歌曲、专辑、艺人的 CRUD 管理，分类体系，搜索发现，播放量统计。会员可上传音乐待审核。

### 3.2 数据结构

```
Song 实体
├── id: Long (PK)
├── title: String                // 歌曲名
├── artistId: Long (FK)          // 主艺人
├── albumId: Long (FK, nullable) // 所属专辑
├── duration: Integer             // 时长（秒）
├── audioUrl: String              // 音频文件路径
├── coverUrl: String              // 封面图路径
├── lyricUrl: String              // LRC 歌词路径（原始文件）
├── lyrics: TEXT                  // 解析后的歌词 JSON
├── genre: String                 // 风格（流行/摇滚/古典等）
├── language: String              // 语种
├── releaseYear: Integer          // 发行年份
├── status: Enum(PENDING, ACTIVE, REJECTED) // 审核状态
├── uploaderId: Long (FK)        // 上传者（会员上传时非空）
├── playCount: Long               // 播放量（定时落库）
├── createdTime: LocalDateTime
└── updatedTime: LocalDateTime

Album 实体
├── id: Long (PK)
├── title: String                 // 专辑名
├── artistId: Long (FK)           // 所属艺人
├── coverUrl: String              // 封面
├── description: TEXT             // 简介
├── releaseDate: LocalDate        // 发行日期
├── createdTime: LocalDateTime
└── updatedTime: LocalDateTime

Artist 实体
├── id: Long (PK)
├── name: String                  // 艺人名
├── avatar: String                // 头像
├── bio: TEXT                     // 简介
├── country: String               // 国籍/地区
├── createdTime: LocalDateTime
└── updatedTime: LocalDateTime
```

### 3.3 接口方法

| 方法 | URL | 说明 | 鉴权 |
|------|-----|------|------|
| `listSongs` | GET /api/songs | 歌曲列表(分页/搜索/筛选) | 可选 |
| `getSongDetail` | GET /api/songs/{id} | 歌曲详情 | 可选 |
| `getLyrics` | GET /api/songs/{id}/lyrics | LRC 歌词 JSON | 可选 |
| `reportPlay` | POST /api/songs/{id}/play | 播放计数上报 | JWT |
| `getHotSongs` | GET /api/songs/hot | 热门歌曲 | 可选 |
| `searchSongs` | GET /api/songs?q=xxx | 模糊搜索（歌曲/艺人/专辑） | 可选 |
| `listAlbums` | GET /api/albums | 专辑列表 | 可选 |
| `getAlbumDetail` | GET /api/albums/{id} | 专辑详情(含歌曲) | 可选 |
| `listArtists` | GET /api/artists | 艺人列表 | 可选 |
| `getArtistDetail` | GET /api/artists/{id} | 艺人详情+作品 | 可选 |
| `memberUpload` | POST /api/members/upload | 会员上传音乐 | JWT+Vip |
| `myMusic` | GET /api/members/music | 我的音乐库 | JWT |

### 3.4 实现逻辑

#### 3.4.1 搜索逻辑

```
① 接收 q 参数（歌曲名/艺人名/专辑名）
② LIKE %q% 模糊匹配三张表
③ 按相关性排序：标题匹配 > 艺人匹配 > 专辑匹配
④ 分页返回（默认 20 条/页）
⑤ Redis 缓存搜索结果（TTL 60s）
```

#### 3.4.2 播放量统计

```
① 每次播放请求 → Redis INCR play_count:buffer:{song_id}
② 定时任务每 5 分钟执行：
   - SMEMBERS play_count:buffer 取出所有 key
   - batch UPDATE song SET play_count = play_count + count
   - DEL play_count:buffer
③ 热门榜单从 Redis SortedSet 按 score 降序取
```

#### 3.4.3 会员上传流程

```
① 校验当前用户会员状态（未过期）
② 校验文件类型（MP3）+ 文件大小（≤50MB）
③ 写入本地文件系统 /data/music/audio/{yyyy/mm}/
④ 写入 song 记录，status = PENDING
⑤ 通知管理员有新上传待审核
```

### 3.5 边界条件

| 边界 | 处理方式 |
|------|---------|
| 搜索无结果 | 返回空列表，不是错误 |
| 搜索关键词为特殊字符 | SQL 转义或 Hutool 过滤 |
| 歌曲不存在 | 400, code=2001 |
| 专辑包含 0 首歌曲 | 正常返回专辑信息 + 空歌曲列表 |
| 会员上传非 MP3 文件 | 400, code=4001 |
| 非会员调用上传 | 400, code=5002 |
| 批量导入 100+ 首歌 | 异步处理，后端队列 |

### 3.6 测试用例

| 编号 | 测试场景 | 预期 |
|------|---------|------|
| UT-09 | 搜索歌曲名命中 | 返回正确歌曲列表 |
| UT-10 | 搜索无结果 | 返回空列表 |
| UT-11 | 获取热门歌曲 | 按播放量降序 |
| UT-12 | 会员上传 MP3 | status = PENDING |
| UT-13 | 非会员上传 | 403 code=5002 |
| UT-14 | 查询专辑详情 | 含专辑信息 + 歌曲列表 |
| UT-15 | 播放量上报 3 次后 | 检查计数累加 |

## 4. 音乐播放模块

### 4.1 功能概述

全局音频播放器，LRC 歌词同步，播放队列管理，播放模式切换，播放进度记忆，最近播放记录。

### 4.2 数据结构

```
// 后端无独立实体，使用 Redis 存储状态

Redis 数据结构：
├── cache:lyrics:{songId} → JSON [{time, text}]       // 歌词缓存
├── play_history:{userId} → ZSET [{songId, playTime}] // 最近播放（最新100条）
├── progress:{userId}:{songId} → String "45000"       // 播放进度（ms）
└── play_count:buffer → Hash {songId → count}        // 播放计数缓冲

// 前端 Pinia Store 状态
PlaybackState {
    currentSong: Song | null       // 当前歌曲
    queue: Song[]                  // 播放队列
    queueIndex: number             // 队列当前位置
    mode: PlayMode                 // SEQUENCE | REPEAT_ONE | SHUFFLE | LOOP
    volume: number                 // 音量 0-1
    progress: number               // 当前进度 ms
    duration: number               // 总时长 ms
    isPlaying: boolean
    isMuted: boolean
}
```

### 4.3 接口方法

| 方法 | URL | 说明 | 鉴权 |
|------|-----|------|------|
| `getHistory` | GET /api/history | 最近播放记录 | JWT |
| `clearHistory` | DELETE /api/history | 清除播放记录 | JWT |
| `reportProgress` | POST /api/progress | 上报播放进度 | JWT |
| `getProgress` | GET /api/progress/{songId} | 获取播放进度 | JWT |

### 4.4 实现逻辑

#### 4.4.1 全局播放器（前端）

```
① Pinia 状态管理 playbackStore
② Audio 实例全局单例，不随页面切换销毁
③ 组件结构：
   PlayerBar.vue      — 底部固定播放条
   DiscAnimation.vue  — 黑胶碟片旋转
   LyricsPanel.vue    — 歌词面板
   PlayQueue.vue      — 播放队列抽屉
④ 跨页面保持播放状态（切换页面不中断音乐）
```

#### 4.4.2 LRC 歌词同步

```
后端：
① 读取 LRC 文件内容
② 正则解析：[mm:ss.xx]歌词文本
③ 转换为 JSON：[{time: 120.5, text: "歌词"}, ...]
④ 缓存到 Redis cache:lyrics:{songId}（TTL 1800s）

前端：
① requestAnimationFrame 动画循环驱动
② 根据 Audio.currentTime 找到当前歌词索引
③ 高亮当前行，提前 2 行滚动容器
④ 支持点击歌词行跳转到对应进度
```

#### 4.4.3 播放队列

```
① 不同入口触发行为：
   - 点击歌曲 → 替换当前歌曲，原歌曲放"下一首"
   - 点击歌单 → 清空队列，用歌单所有歌曲填充
   - 点击"下一首播放" → 插入到队列当前位置+1
② 播放模式影响"下一首"行为：
   - SEQUENCE: 播完自动停止（列表末尾）
   - LOOP: 播完从队列头重新开始
   - REPEAT_ONE: 循环当前歌曲
   - SHUFFLE: 随机取队列中未播放的下一首
③ 队列持久化到 localStorage，刷新页面不丢失
```

#### 4.4.4 进度记忆

```
① 前端每 10 秒调用 POST /api/progress 上报 currentTime
② 后端写入 Redis progress:{userId}:{songId}（TTL 7天）
③ 进入歌曲时检查是否有进度记录
④ 有 → 弹窗提示"检测到上次听到 xx:xx，是否续播？"
⑤ 用户选择"是" → audio.currentTime = savedProgress
```

### 4.5 边界条件

| 边界 | 处理方式 |
|------|---------|
| 音频文件加载失败 | 显示加载错误，自动播放下首 |
| 网络切换/断网 | Audio 触发 error，暂停并提示 |
| 队列为空时点下一首 | 不做任何操作 |
| SHUFFLE 模式所有歌曲已播完 | 重置已播放标记，重新洗牌 |
| 歌词文件不存在 | 歌词面板显示"暂无歌词" |
| 同时从多端播放同一账号 | 以最后一次进度上报为准 |

### 4.6 测试用例

| 编号 | 测试场景 | 预期 |
|------|---------|------|
| UT-16 | 解析标准 LRC 文件 | 正确的 JSON 时间轴 |
| UT-17 | 播放模式切换 LOOP | 播完队列尾自动跳到头 |
| UT-18 | SHUFFLE 模式不重复 | 所有歌曲播完一轮才重复 |
| UT-19 | 上报进度 3 次后查询 | 返回最近一次上报值 |
| UT-20 | 队列为空 | 播放按钮无响应 |
| UT-21 | 最近播放满 100 条 | 新记录淘汰最旧的 |

## 5. 交互模块

### 5.1 功能概述

评论系统、收藏/喜欢功能、歌单 CRUD，是平台社交互动核心。

### 5.2 数据结构

```
Comment 实体
├── id: Long (PK)
├── userId: Long (FK)
├── targetType: Enum(SONG, PLAYLIST)  // 评论对象类型
├── targetId: Long                      // 评论对象 ID
├── content: String(VARCHAR 500)        // 评论内容（已转义）
├── parentId: Long (nullable)           // 回复的评论 ID（楼中楼）
├── status: Enum(VISIBLE, DELETED)     // 可见状态
├── createdTime: LocalDateTime
└── updatedTime: LocalDateTime

Favorite 实体
├── id: Long (PK)
├── userId: Long (FK)
├── targetType: Enum(SONG, PLAYLIST, ALBUM)
├── targetId: Long
├── createdTime: LocalDateTime
└── UNIQUE(userId, targetType, targetId)  // 防重复收藏

Likes 实体  (与 Favorite 结构类似，但语义不同：红心轻量标记)
├── id: Long (PK)
├── userId: Long (FK)
├── targetType: Enum(SONG, PLAYLIST, COMMENT)

├── createdTime: LocalDateTime

Playlist 实体

PlaylistSong 关联实体
├── id: Long (PK)
├── playlistId: Long (FK)
├── songId: Long (FK)
├── sortOrder: Integer                // 排序序号
├── addedTime: LocalDateTime
└── UNIQUE(playlistId, songId)
```

### 5.4 实现逻辑

#### 5.4.1 评论系统

```
① 发表评论：
   - XSS 过滤（HTML 标签转义）
   - 敏感词过滤（敏感词库匹配替换为 ***）
   - 校验 targetType + targetId 有效性

### 6.1 功能概述

### 6.2 数据结构

```
// Redis 实时数据（不使用 MySQL 存房间状态）
Redis Hash room:{roomId}:
├── name: String

RoomMessage 实体
├── id: Long (PK)
├── roomId: Long

### 6.3 接口方法

| 方法 | URL | 说明 | 鉴权 |
|------|-----|------|------|

### 6.4 实现逻辑

#### 6.4.1 WebSocket 连接与鉴权

```
① 前端建立 WebSocket：new WebSocket("ws://host/ws/room/{roomId}?token=xxx")
② 后端 WebSocket 拦截器解析 token：
   - 无效 → 关闭连接，返回 4001
   - 有效 → 将 userId 注入 session 属性
③ 连接建立后，加入 Redis Set room:{roomId}:members
④ 心跳：客户端每 30s 发送 HEARTBEAT，服务端 60s 未收到则主动断开
⑤ 断线重连：客户端指数退避（1s/2s/4s/8s...max 30s）
```

#### 6.4.2 播放同步

```
房主操作流程：
① 房主点击播放/暂停/切歌/拖动进度
② 前端发送 WebSocket message type=SYNC
③ 服务端收到后：
   - 更新 Redis Hash room:{roomId} 中的状态
   - 发布到 Redis Channel room_sync:{roomId}
④ Redis Pub/Sub 监听器收到消息
⑤ 遍历该房间的所有 WebSocket 连接
⑥ 广播 SYNC 消息给所有成员（除发送者）
⑦ 成员前端收到后同步 Audio 状态
```

#### 6.4.3 文字聊天

```
① 用户发送 WebSocket message type=CHAT, payload={content}
② 服务端校验内容长度（≤500）、频率限制（每秒1条）
③ 广播 CHAT 消息给房间所有成员
④ 消息异步写入 MySQL RoomMessage 表（保留最近 50 条）
```

#### 6.4.4 语音通讯（简化版）

```
① 按住录音按钮 → 浏览器录音（MediaRecorder API）
② 录音结束 → 生成 Blob → 上传到 POST /api/files/upload
③ 上传成功后 → 发送 WebSocket message type=VOICE, payload={url, duration}
④ 服务端广播 VOICE 消息
⑤ 成员前端收到后 → 创建 Audio 播放
```

### 6.5 边界条件

| 边界 | 处理方式 |
|------|---------|
| 房间已满人 | 返回 3002(ROOM_FULL) |
| 加入需要密码的房间 | 要求先验证密码 |
| 房主掉线 | 保留房主身份 60 秒，超时自动移交 |
| 房间全员离开 | 自动解散，清理 Redis |
| 同时两个房主操作 | 以 Redis 时间戳较新的为准 |
| 语音文件过大（>5MB） | 限制单条语音 60 秒，约 500KB |

## 11. 安全与审计模块

### 11.1 功能概述

接口限流、XSS 过滤、SQL 注入防护、操作审计。

### 11.2 实现逻辑

#### 11.2.1 接口限流 @RateLimit

```
@Target(METHOD)
@Retention(RUNTIME)
public @interface RateLimit {
    int count() default 60;       // 60 次
    int time() default 60;        // 60 秒
    String key() default "";      // 限流 key，默认 userId + URI
}

// AOP 实现（基于 Redis + Lua 脚本）
① 解析注解，构建限流 key：ratelimit:{userId}:{uri}
② 执行 Lua 脚本（原子操作）：
   - INCR key
   - 如果是第一次，EXPIRE key time
   - 返回当前计数
③ 当前计数 > count → 抛出 TOO_MANY_REQUESTS
```

#### 11.2.2 XSS 过滤

```
① 所有用户输入（评论/昵称/简介）存储前进行 HTML 转义
② 使用 Hutool HtmlUtil.filter() 过滤标签
③ 仅允许安全的文本格式
```

#### 11.2.3 操作审计 @OperationLog

```
① 自定义注解 + AOP
② 记录：操作人、操作类型、目标对象、IP、请求参数、结果
③ 异步写入（@Async），不阻塞主流程
④ 提供查询接口供管理员查看
```

## 附录：API 状态码速查

### 业务码（code 字段）

| 范围 | 模块 | 示例 |
|------|------|------|
| 200 | 通用成功 | 200 操作成功 |
| 1xxx | 用户模块 | 1001 邮箱已注册, 1002 账号或密码错误 |
| 2xxx | 音乐模块 | 2001 歌曲不存在, 2005 歌曲已存在 |
| 3xxx | 歌房模块 | 3001 房间不存在, 3003 不是房主 |
| 4xxx | 文件模块 | 4001 文件类型不支持, 4002 文件过大 |
| 5xxx | 会员模块 | 5001 会员过期, 5002 需会员 |
| 9xxx | 通用错误 | 400 参数错误, 401 未登录, 403 无权, 404 不存在 |

### HTTP 状态码（http status）

| 状态码 | 说明 | 场景 |
|--------|------|------|
| 200 | 正常响应 | 所有业务码都在 200 响应体中 |
| 401 | 未认证 | Token 缺失/过期/黑名单 |
| 403 | 无权限 | 非管理员访问管理接口 |
| 404 | 资源不存在 | 路由 404、静态资源 404 |
| 429 | 请求过多 | 被限流拦截 |
| 500 | 系统异常 | 数据库宕机、空指针等未预期错误 |
