## 1. 项目概述

在线音乐平台是一个全栈 Web 应用，提供音乐发现、播放、社交互动（歌房/评论/收藏）、会员体系、管理后台等完整功能。

| 维度 | 说明 |
|------|------|
| 后端端口 | `8888` |
| 前端端口 | `5173`（开发）/ 打包后由后端 `8080` 代理 |
| 前端代理 | Vite 开发服务器将 `/api` 和 `/data` 代理到后端 `8888` |
| 数据库 | MySQL `music_db` |
| 缓存 | Redis（Token 黑名单、歌房状态、限流计数器、播放计数缓冲、排行榜） |
| 文件存储 | 本地文件系统 `{project_dir}/data/music/` |
| 里程碑进度 | **全功能上线 100%** |

### 4.3 环境变量配置

生产环境请设置以下环境变量（覆盖 application.yml 默认值）：

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `MYSQL_PASSWORD` | 数据库密码 | `1234567` |
| `REDIS_HOST` | Redis 地址 | `localhost` |
| `REDIS_PASSWORD` | Redis 密码 | （空） |
| `MUSIC_JWT_SECRET` | JWT 签名密钥 | 开发默认值 |

| `SPRING_PROFILES_ACTIVE` | 激活配置 | `dev` |

### 4.4 启动后端

```bash
# 方式一：Maven 直接运行
cd E:\07_Data\IdeaProjects\music
mvn spring-boot:run

# 方式二：打包后运行
mvn clean package -DskipTests
java -jar target/music-0.0.1-SNAPSHOT.jar

# 方式三：指定环境
mvn spring-boot:run -Dspring-boot.run.profiles=prod
```

```bash

npm run dev      # 开发模式 (端口 5173)
```

| **歌曲详情** `/songs/:id` | DiscAnimation 黑胶动画、播放/收藏/点赞/举报/添加到歌单、歌词同步、评论 | `GET /songs/:id`, `POST /songs/:id/play`, `POST/DELETE /favorites/song/:id`, `POST/DELETE /likes/song/:id`, `GET /playlists/mine`, `POST /playlists/{id}/songs`, `POST /reports` |

## 7. 核心模块详解

### 7.1 认证与鉴权

```
请求 → JwtInterceptor → 检查白名单 → 需要认证？ → 校验 access_token → 通过 → 控制器
                                            ↓ 否                       ↓ 失败
                                        直接放行                    401 + 前端自动 refresh_token
```

- **access_token**: 2 小时过期，存 localStorage
- **refresh_token**: 7 天过期，自动静默刷新
- 退出登录时将 access_token 加入 Redis 黑名单
- 密码使用 BCrypt 加密存储（strength=4，单次校验 ~50ms）
- 角色体系：`USER`（普通用户） / `VIP`（会员） / `ADMIN`（管理员）
- **白名单策略**：精确路径白名单放行所有方法；GET 请求前缀白名单仅放行 GET，POST/PUT/DELETE 需认证
- **密码重置**：6 位数字验证码存储 Redis（5 分钟 TTL），验证通过后可重置密码

### 7.2 播放系统

- 全局唯一的 `Audio` 实例，跨页面不中断
- 4 种播放模式：顺序(SEQUENCE)、列表循环(LOOP)、单曲循环(REPEAT_ONE)、随机(SHUFFLE)
- 播放队列持久化到 `localStorage`
- 每 10 秒自动上报播放进度到 Redis（7 天 TTL）
- 播放历史记录到后端（最多 100 条）
- 续播提示：播放歌曲时检查上次进度 >5 秒则询问是否续播

### 7.3 排行榜系统

```
reportPlay() → Redis ZAdd/IncrementScore
  ├── ranking:daily:{yyyyMMdd}    ← 当日播放
  ├── ranking:weekly:{yyyy'w'ww}  ← 当周播放
  ├── ranking:monthly:{yyyyMM}    ← 当月播放
  └── ranking:alltime             ← 全量播放

GET /api/rankings?type=daily → ZRevRange → SongDetail → SongVO(含rank)
```

### 7.4 文件上传

```
上传 → 检查类型 → MIME校验 → 大小校验 → 魔数校验 → UUID重命名 → 日期目录 → 写入磁盘 → 返回URL
```

- 文件存储根目录：`{project_dir}/data/music/`
- 静态资源映射：`/api/files/**` → `file:{project_dir}/data/music/`
- 文件命名：`{type}/{yyyy}/{MM}/{uuid}.{ext}`
- 语音特殊路径：`voice/{roomId}/{uuid}.webm`

### 7.5 通知系统（实时推送）

```
通知触发 → DB INSERT → Redis Pub/Sub发布
  ├── USER_NOTIFY        → 目标用户 WebSocket 推送
  └── SYSTEM_BROADCAST   → 所有在线用户广播

WebSocket 端点：ws://host/ws/notify?token=xxx
NotifySessionManager：管理 userId → Set<WebSocketSession> 映射
```

### 7.6 歌房实时通信

```
创建房间 → 用户加入 → WebSocket 连接
                      ├── CHAT 消息（实时聊天）
                      ├── SYSTEM 消息（成员进出）
                      ├── SYNC 消息（播放同步）
                      └── HEARTBEAT（30秒心跳）
```

- 基于 Spring WebSocket + Redis Pub/Sub 实现跨实例消息广播
- 房间状态：WAITING / PLAYING / PAUSED
- 房主权限：踢出成员、转让房主、解散房间
- 房间类型：公开 / 私密（密码保护）

### 7.7 限流系统（@RateLimit）

```java
@RateLimit(key = "register", max = 5, time = 60)  // 60秒内最多5次
```

- 基于 Redis + Lua 脚本实现滑动窗口计数
- 默认限流策略：60 秒内最多 30 次
- 限流响应：HTTP 429 + 错误码 3001

### 7.8 审计日志（@OperationLog）

```java
@OperationLog(value = "封禁/解封用户", targetType = "USER")
```

- `@Around` 环绕通知，拦截所有带注解的方法
- 异步写入（`@Async`），不阻塞主流程
- 自动记录：操作人、操作类型、目标对象/ID、请求参数、响应结果、IP 地址
- 自动推断 targetType（从 action 文本中匹配"用户"/"歌曲"等关键字）
- 自动推断 targetId（从方法参数中找 Long/Integer 类型）
- IP 获取顺序：`X-Forwarded-For` → `X-Real-IP` → `getRemoteAddr()`

### 7.9 定时任务

- **PlayCountFlushTask**: 每 30 秒将 Redis 中的播放计数批量刷新到 MySQL（使用 CASE WHEN 单语句）

### 10.4 技术要点记录

- **Java 17** 新特性：`record`、`switch` 表达式、`text block`
- **Spring Boot 3.2**：Jakarta EE 命名空间、虚拟线程（可用）、GraalVM native-image（可用）
- **AOP 类名冲突解决**：`OperationLog` 既是注解也是实体，使用 FQN `com.example.music.entity.OperationLog`
- **WebSocket 鉴权**：通过 URL query 参数传递 Token，在 Interceptor 中验证
- **Snowflake ID**：41bit 时间戳 \| 10bit 工作节点 \| 12bit 序列号，带时钟回滚保护
- **排行榜实现**：基于 Redis ZSet，通过播放上报自动更新四个维度（日/周/月/总）
- **通知推送流程**：Service → DB → Redis Pub/Sub → NotifySessionManager → WebSocket 推送
- **文件路径**：当前使用绝对路径 `E:/07_Data/...`，建议迁移至 `@Value("${music.file.upload-dir}")` 外部化配置
