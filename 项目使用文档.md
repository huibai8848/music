## 1. 项目概述

在线音乐平台是一个全栈 Web 应用，提供音乐发现、播放、社交互动（歌房/评论/收藏）、会员体系、管理后台、排行榜等完整功能。

| 维度 | 说明 |
|------|------|
| 后端端口 | `8888` |
| 前端端口 | `5173`（开发）/ 打包后由后端 `8080` 代理 |
| 前端代理 | Vite 开发服务器将 `/api`、`/data`、`/ws` 代理到后端 `8888` |
| 数据库 | MySQL `music_db`（19 张表） |
| 缓存 | Redis（Token 黑名单、歌房状态、限流、排行榜 ZSet、计数缓冲、进度记忆） |
| 文件存储 | 本地文件系统 `{project_dir}/data/music/` |
| 实时通信 | WebSocket（歌房聊天+播放同步、通知推送） |

### 4.2 数据库初始化

```bash
# 方式一：使用 DDL 文件批量执行
mysql -u root -p < tmp_ddl.sql
mysql -u root -p < tmp_init.sql

# 方式二：登录后执行
mysql -u root -p
mysql> source tmp_ddl.sql;
mysql> source tmp_init.sql;
```

| `REDIS_PASSWORD` | Redis 密码 | （空） |
| `MUSIC_JWT_SECRET` | JWT 签名密钥 | 开发默认值 |

## 6. 后端 API 总览

### 6.1 认证模块 — `/api/auth`

| 方法 | 路径 | 说明 | 限流 |
|------|------|------|------|
| GET | `/api/auth/public-key` | 获取 RSA 公钥 | — |
| POST | `/api/auth/register` | 注册新用户 | 5/min/IP |
| POST | `/api/auth/login` | 登录（支持 RSA 加密密码） | 10/min/IP |
| POST | `/api/auth/refresh` | 刷新 access_token | — |
| POST | `/api/auth/logout` | 登出（黑名单当前 Token） | — |
| POST | `/api/auth/forgot-password` | 发送密码重置验证码 | 3/300s |
| POST | `/api/auth/reset-password` | 验证码重置密码 | — |

### 6.2 用户模块 — `/api/users`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| GET | `/api/users/me` | 获取当前用户信息 | JWT |
| PUT | `/api/users/me` | 修改个人资料 | JWT |
| PUT | `/api/users/me/password` | 修改密码 | JWT |

| GET | `/api/users/{id}` | 用户公开主页 | 公开 |

### 6.3 歌曲模块 — `/api/songs`

| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| GET | `/api/songs` | 分页列表（支持 genre/language/releaseYear/artistId/albumId 筛选） | 公开 |
| GET | `/api/songs/search` | 模糊搜索（歌曲/艺人/专辑名） | 公开 |
| GET | `/api/songs/hot` | 热门歌曲（play_count 降序） | 公开 |
| GET | `/api/songs/{id}` | 歌曲详情 | 公开 |
| GET | `/api/songs/{id}/lyrics` | LRC 歌词 JSON | 公开 |
| GET | `/api/songs/{id}/categories` | 歌曲分类 ID 列表 | 公开 |
| PUT | `/api/songs/{id}/categories` | 更新歌曲分类 | Admin |
| POST | `/api/songs/{id}/play` | 上报播放次数 | 60/min/用户 |
| POST | `/api/songs` | 创建歌曲 | Admin |

## 7. 核心模块详解

### 7.1 认证与鉴权

```
请求 → JwtInterceptor
  → 检查白名单（精确匹配/GET前缀/静态资源）
  → 需要认证？ → 校验 access_token → 通过 → 注入 RequestContext → 控制器
              ↓ 否（白名单）                              ↓ 失败
           直接放行（可选的 Token 注入）              401 + 前端自动 refresh_token
```

- **access_token**: 2 小时过期，存 sessionStorage
- **refresh_token**: 7 天过期，自动静默刷新
- 退出登录时将 access_token 加入 Redis 黑名单
- 密码使用 BCrypt 加密存储（strength=4）
- 登录密码支持 RSA 加密传输脱敏
- 角色体系：`USER` → `VIP` → `ADMIN`

### 7.2 全局播放器

- 全局唯一的 `Audio` 实例，跨页面不中断
- 4 种播放模式：顺序(SEQUENCE)、列表循环(LOOP)、单曲循环(REPEAT_ONE)、随机(SHUFFLE)
- 播放队列持久化到 `localStorage`
- 每 10 秒自动上报播放进度到 Redis（7 天 TTL）
- 播放历史记录到后端（最多 100 条）
- 续播提示：播放歌曲时检查上次进度 >5 秒则询问是否续播

### 7.3 排行榜系统

```
reportPlay(songId) → Redis ZINCRBY 四个维度的 ZSet：
  ranking:daily:{yyyyMMdd}
  ranking:weekly:{yyyy'w'ww}
  ranking:monthly:{yyyyMM}
  ranking:alltime

GET /api/rankings → ZREVRANGE → 获取 songId → 组装 SongVO（含排名）
```

### 7.4 文件上传

```
上传 → 校验类型 → MIME 校验 → 大小校验 → 魔数校验
  → UUID 重命名 → 按日期分目录 → 写入磁盘 → 返回 URL
```

### 7.5 歌房实时通信

```
创建房间 → WebSocket 连接
  ├── CHAT 消息（实时聊天，广播给所有成员）
  ├── SYSTEM 消息（成员进出通知）
  ├── SYNC 消息（播放同步：播放/暂停/切歌/进度拖动）
  └── HEARTBEAT（30秒心跳，60秒无响应断开）
```

- 基于 Spring WebSocket + Redis Pub/Sub 实现跨实例消息广播
- 房间状态：WAITING / PLAYING / PAUSED
- 房间类型：公开 / 私密（密码保护）
- 支持加载歌单、专辑、收藏歌曲到队列

### 7.6 通知系统

```
通知触发 → 写入 DB → afterCommit → Redis Pub/Sub
  → 用户在线 → WebSocket 实时推送
  → 用户离线 → 下次登录显示未读角标
```

### 7.7 限流系统

```java
@RateLimit(count = 60, time = 60)  // 60 秒内最多 60 次
```

- 基于 Redis + Lua 脚本实现滑动窗口计数
- 限流响应：HTTP 429

### 7.8 审计日志

```java
@OperationLog(value = "封禁/解封用户", targetType = "USER")
```

- AOP 环绕通知 + @Async 异步写入
- 自动记录操作人、操作类型、目标、参数、结果、IP

### 7.9 定时任务

| 任务 | 周期 | 说明 |
|------|------|------|
| PlayCountFlushTask | 30 秒 | Redis 播放计数批量刷入 MySQL（CASE WHEN 单语句） |

### 9.3 请求封装（request.js）

- 自动携带 `Authorization: Bearer {access_token}`
- 401 响应时自动尝试 `refresh_token` 刷新
- 多个并发请求过期时排队等待刷新，仅刷新一次
- 刷新失败则清除登录状态并跳转登录页
